项目亮点

智能算法优化

采用多目标优化算法，结合贪心策略和启发式搜索，实现高效的课程安排，将传统人工排课时间从数周缩短至分钟级别。

实时冲突检测

系统内置强大的约束检查引擎，能够实时检测并解决教师时间、教室容量、课程依赖等各类冲突，确保排课结果的可行性。

可视化管理界面

直观的课表展示和拖拽式操作，支持快速调整和实时预览，让教务管理变得简单高效，提升用户体验。

数据分析报告

提供详细的资源利用率分析、教师工作量统计、教室使用效率等数据报告，为教务决策提供科学依据。

需求分析文档

项目背景

当前许多院校的排课工作仍依赖人工或半自动化的Excel表格进行，效率低下、易出错，且难以应对复杂的排课约束和动态调整需求。本项目旨在开发一套现代化的排课信息系统。

项目目标

提高排课效率

大幅缩短排课周期，减轻教务管理员的工作负担。通过自动化处理，将排课时间从数周缩短至数分钟。

减少课程冲突

通过智能算法和规则校验，最大限度地避免时间、地点、教师等资源冲突。

优化资源利用

合理分配教室、实验室、自习室等教学资源，提高其使用率。

功能需求

用户管理模块

  • 支持创建不同角色的用户账号(教师、学生)
  • 用户通过学号/工号和密码登录系统
  • 提供密码找回/重置功能
  • 为不同角色分配不同的操作权限

基础数据管理

  • 教室信息管理：教室编号、容量、类型
  • 教师信息管理：工号、姓名、职称、禁排时间
  • 课程信息管理：课程编号、名称、学分、类型
  • 专业/班级信息管理：专业代码、专业名称、人数

排课规则设置

  • 时间规则：午休/节假日禁排、连排课时限制
  • 教师规则：每日/周最大课时数、避免连续上课
  • 教室规则：特定课程安排在特定类型教室
  • 课程规则：互斥课程不能安排在同一天

系统设计文档

系统架构

采用前后端分离的现代化架构设计，确保系统的可扩展性和维护性。

前端架构

Vue.js + Element UI，提供响应式、现代化的用户界面

后端架构

Flask + SQLAlchemy + MySQL，高性能的API服务

排课引擎

智能排课算法，多目标优化，支持复杂约束

核心模块设计

排课算法流程

  1. 初始化：读取所有课程、教师、教室数据
  2. 约束条件处理：应用所有排课规则
  3. 资源分配：使用贪心算法进行初始分配
  4. 冲突检测与优化：多目标优化资源分配
  5. 结果输出：生成无冲突课表

数据库设计

主要数据库表包括：用户表、课程表、教师表、教室表、班级表、课表表、权限表等。采用规范化设计，确保数据一致性和完整性。

接口设计

  • 用户认证接口：登录、注册、权限验证
  • 数据管理接口：CRUD操作接口
  • 排课接口：排课规则设置、排课执行、结果查询
  • 查询接口：课表查询、资源查询
